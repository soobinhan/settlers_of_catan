module Definition: sig .. end

Game Representation


type color =
| Blue
| Red
| Orange
| White
This is the forward cyclic order of player turns
type terrain =
| Hill
| Pasture
| Mountain
| Field
| Forest
| Desert
The six terrain types on the board hexes. Each produces a corresponding resource when the hex's dice number is rolled.
type resource =
| Brick
| Wool
| Ore
| Grain
| Lumber
The resource types produced by the above terrains, respectively. Desert does not produce any resource
type card =
| Knight
| VictoryPoint
| RoadBuilding
| YearOfPlenty
| Monopoly
The types of development cards available in the game.
type roll = int
Dice roll values. Valid for 2 - 12 inclusive.

Hex list index. Valid for 0-18 inclusive.

type piece = int
Point list index. Valid for 0 - 53 inclusive.
type point = int
Maritime trade ratio. Valid for 2, 3, or 4
type ratio = int 
type line = point * point
Valid for two adjacent points. (p1, p2) is line-equal to (p2, p1)

One tile on the map. roll is the dice number that will produce resources

type hex = terrain * roll 
type cost = int * int * int * int * int
In resource enumerated order (B,W,O,G,L)

Number of each resource a player has available in B,W,O,G,L order

type inventory = cost 
type cards =
| Hidden of int
| Reveal of card list (*Represents a player's hand*)
All card lists in the game are encoded in this type so that players cannot look at cards they shouldn't be able to access
type hand = inventory * cards

TROPHIES: properties of individual players


type knights = int
Number of knights the player has played thus far

True if the player currently has the longest road

type longestroad = bool
True if the player currently has the largest army (most knights played)
type largestarmy = bool
The three trophies the player can own (property of an individ. player
type trophies = knights * longestroad * largestarmy 
type player = color * hand * trophies
Player's color, cards they own, and trophies they have. One of four "big" components in the overall game state.
type portresource =
| Any
| PortResource of resource (*A port provides a specific trade ratio (e.g. ratio 4 means a 4:1 trade) and services the two points along its line.*)
Ports can either offer a trade ratio for a specific resource, or can offer a generic ratio for any resource
type port = line * ratio * portresource 
type settlement =
| Town
| City
Settlement type.
type intersection = (color * settlement) option
Some (c,s) if the settlement s of color c exists on the board. Each intersection is indexed by a point

A road built by a player of a specific color, across a specific line on the board

type road = color * line 
type map = hex list * port list
The static map: 19 tiles, 9 ports standard. Lists are in the indexed order given by the game board in the writeup.

The variable board: 53 intersections standard, 0 initial roads

type structures = intersection list * road list
Deck of cards not yet drawn.
type deck = cards
Discard pile of cards played in discarded order.
type discard = card list
The hex currently occupied by the robber. The robber is always on a piece
type robber = piece 
type board = map * structures * deck *
       discard * robber
Keeps track of everything on the physical gameboard. One of four "big" components of the overall game state.
type trade = color * cost * cost
(id * cost1 * cost2) is a (proposed) exchange of cost1 from active for cost2 with id. trades can only be initiated by the active player as part of their turn.
type turn = {
   active : color; (*The color of the active player*)
   dicerolled : roll option; (*Some roll if dice were rolled this turn*)
   cardplayed : bool;
   cardsbought : cards; (*Any cards a player buys during their turn are stored in turn. cardsbought and are only transfered to the player's hand at the END of the turn*)
   tradesmade : int; (*Number of trades made this turn*)
   pendingtrade : trade option; (*A trade that's just been proposed, but not yet accepted/rejected.*)
}
Current turn state keeps track of what has happened during the turn and is renewed when a new turn begins
type request =
| InitialRequest
| RobberRequest
| DiscardRequest
| TradeRequest
| ActionRequest
Five different move types that the game can request a player to make
type next = color * request
Color of the next player to be prompted * type of move requested this is updated after every move, for the following move.
type state = board * player list * turn * next
ALL the information needed to represent the current game state *

Moves


type maritimetrade = resource * resource
Resource type sold * resource type bought
type build =
| BuildRoad of road
| BuildTown of point
| BuildCity of point
| BuildCard
Can only build on your turn
type robbermove = piece * color option
The hex the robber will move to, and color of an adjacent player to rob, if one is available
type playcard =
| PlayKnight of robbermove
| PlayRoadBuilding of road * road option
| PlayYearOfPlenty of resource * resource option
| PlayMonopoly of resource
RoadBuilding, YearOfPlenty is also valid with only one road, resource
type action =
| RollDice
| MaritimeTrade of maritimetrade
| DomesticTrade of trade
| BuyBuild of build
| PlayCard of playcard
| EndTurn
The types of actions a player can perform in response to an ActionRequest
type move =
| InitialMove of line (*(p1 * p2) as line places a town at p1 and a road from p1 to p2*)
| RobberMove of robbermove
| DiscardMove of cost (*Number of each resource the player wishes to discard, in B,W,O,G,L order*)
| TradeResponse of bool (*true to accept the trade, false to reject.*)
| Action of action
The move types a player can make, in response to InitialRequest, RobberRequest, DiscardRequest, TradeRequest, and ActionRequest, respectively

Winning


type 'a outcome = color option * 'a
Some c if player c has won; None if game is still in progress
