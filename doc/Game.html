val init_game : unit -> game
Create a brand new initial game state. This is non-deterministic. You may change this to experiment with alternate generators, such as gen_random_initial_state, to test your bot's decision making in varied circumstances.
val handle_move : game -> Definition.move -> game Definition.outcome
handle_move g m applies the next-anticipated move, m, to the game, g. If m is an invalid move in context, a random minimum viable move is chosen instead. This will produce the outcome, which indicates whether someone has won, and the updated game after the move has taken place.
val presentation : game -> game
presentation g modifies a game instance g such that passing it to the next-expected player will not reveal any information that should not be accessable to that player.
